Why Canopy?

Before

The CLI Workflow

$ git worktree list $ git worktree add ../feature-x -b feature-x $ cd ../feature-x $ claude

Switching contexts means remembering commands, navigating directories, and managing multiple terminal windows.

After

The Canopy Way

Click → Done

One click to create a worktree. One click to open it in your terminal. That's it.

Features

Worktree Management

  • Create, list, and remove worktrees instantly
  • Visual dirty status indicators
  • Smart confirmation only when needed
  • Support for multiple repositories

Terminal Integration

  • One-click launch into your terminal
  • Supports Warp and iTerm2
  • Perfect for Claude Code sessions
  • Opens directly in worktree directory

GitHub Integration

  • PR status badges at a glance
  • CI checks status: passing, failing, or pending
  • Merge conflict warnings when PRs can't be merged
  • Powered by GitHub CLI (gh)

Setup & Run

Configure Canopy to run your apps and access workspace links

Run Command

Set a command to run when you click the play button on a worktree.

  1. Open Settings (gear icon)
  2. Select your repository
  3. Edit the Run command field
npm run dev python manage.py runserver

Post-Create Hook

Run a command automatically after creating a new worktree.

  1. Open Settings
  2. Edit the Post-create command field
npm install bundle install

Workspace Links

Add a clickable URL to any worktree for quick access to dev servers or other resources.

  1. Create a file named CANOPY_URL.txt in your worktree
  2. Add a single line with your URL
  3. Click the link icon in Canopy to open it
echo "http://localhost:5173" > CANOPY_URL.txt

Pro tip: If using different ports for each workspace, use your post-create hook to generate this file.

Getting Started

Requirements

  • macOS 14+ (Sonoma or later)
  • Git installed
  • Optional: Warp or iTerm2 for terminal integration
  • Optional: GitHub CLI (gh) for PR features

Installation

  1. 1
    Download the DMG

    Get the latest release from GitHub

  2. 2
    Drag to Applications

    Open the DMG and drag Canopy to your Applications folder

  3. 3
    First Launch (Unsigned App)

    Since Canopy isn't signed with an Apple Developer certificate, macOS will block it initially. Here's how to open it:

    1. Double-click Canopy in Applications. macOS will show a warning.
    2. Open System Settings, then go to Privacy & Security.
    3. Scroll down to find the "Canopy was blocked" message.
    4. Click Open Anyway and confirm.

    You only need to do this once. After that, Canopy will open normally.
